# i-m-senior-developer — Plugin Marketplace

Монорепозиторий плагинов для Claude Code. Распространяется через GitHub.

## Версионирование плагинов

При изменении любого плагина **обязательно бампнуть `version`** в его `.claude-plugin/plugin.json`. Без этого пользователи не получат обновления — Claude Code кеширует плагины по версии.

Используй **Semantic Versioning** (semver.org):
- **MAJOR** (1.0.0 → 2.0.0) — breaking changes: удалены/переименованы команды, агенты, изменён формат
- **MINOR** (0.1.0 → 0.2.0) — новая функциональность: добавлены команды, агенты, скиллы
- **PATCH** (0.1.0 → 0.1.1) — исправления: баги, опечатки, уточнения в промптах

## Релизный workflow

1. Внести изменения в плагин(ы)
2. Бампнуть `version` в `plugin.json` каждого изменённого плагина
3. Коммит + тег: `git tag vX.Y.Z`
4. Push: `git push && git push --tags`
