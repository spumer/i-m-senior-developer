# I'm Senior Developer

Набор плагинов для Claude Code, в которых собран мой опыт разработки.

Это не фреймворк и не методология ради методологии. Это рабочие принципы, выросшие из практики. Они работают потому что отражают реальность, а не потому что так написано в книге.

## Философия

Программирование — это отражение реальности. Решая задачи, мы моделируем мир. А значит то, что работает в коде — работает и в жизни. И наоборот.

Принцип — это то, что работает везде. Если есть исключение — это не принцип. Принципов не может быть много. Их ровно столько, сколько нужно.

Простота — это не когда нечего добавить, а когда нечего убрать. Надежность — это не когда всё обработано, а когда ничего не спрятано. Понятность — это не когда всё задокументировано, а когда документация не нужна.

Главный враг — не сложность. Главный враг — иллюзия простоты. Проглоченное исключение выглядит как стабильность. Возвращенный None выглядит как обработка ошибки. Логирование без re-raise выглядит как мониторинг. Но за каждой иллюзией — потерянные данные, которые никто никогда не найдет.

Поэтому система должна падать. Честно и громко. Fail-fast — это не про хрупкость. Это про честность. Лучше упасть сейчас и узнать о проблеме, чем тихо потерять данные и узнать через месяц. Или никогда.

Изменения постоянны. Код, который ты пишешь сегодня — завтра придется менять. Поэтому каждое решение должно не только решать задачу, но и делать следующее изменение дешевле. Это не про абстракции "на будущее". Это про ясность, которая снижает стоимость любого изменения.

Ничто ниоткуда не берется и никуда не пропадает. Сложность тоже. Если ты спрятал её в одном месте — она вылезет в другом. Поэтому лучше явная сложность в одном месте, чем скрытая во всех.

## Что внутри

### [functional-clarity](plugins/functional-clarity/)

22 принципа Функциональной ясности. Философия написания надежного, простого и понятного кода. Fail-fast архитектура, запрет Error Hiding, минимальные изменения, явные зависимости.

Загружается при старте сессии. Полный набор принципов доступен через skill.

### [tdd-master](plugins/tdd-master/)

TDD методология по Кенту Беку и дяде Бобу. Red-Green-Refactor, FIRST принципы, паттерны тестирования для pytest и Django.

Два плагина дополняют друг друга:
- **functional-clarity** — КАК писать код
- **tdd-master** — КАК писать тесты

## Установка

### 1. Подключить маркетплейс

Внутри сессии Claude Code:

```
/plugin marketplace add spumer/i-m-senior-developer
```

### 2. Установить плагины

```
/plugin install functional-clarity@i-m-senior-developer
/plugin install tdd-master@i-m-senior-developer
```

### Локальная разработка

```bash
claude --plugin-dir plugins/functional-clarity --plugin-dir plugins/tdd-master
```
