## Тестирование

**Error Hiding антипаттерн**: В тестах используйте `pytest.fail()` вместо `raise TimeoutError()`. Обычные исключения подвержены **Error Hiding** (поглощению исключений) — антипаттерну, когда промежуточные слои кода ловят исключения и "проглатывают" их без надлежащей обработки или логирования. Это создает ложное ощущение стабильности при потере критической диагностической информации.

```python
# Правильно - тест явно падает с понятным сообщением
if not event.wait(timeout=2.0):
    pytest.fail(f'Test design error: timeout waiting for {event_key=}')

# Неправильно - исключение может быть поглощено
if not event.wait(timeout=2.0):
    raise TimeoutError(f'Test design error: {event_key=}')
```
